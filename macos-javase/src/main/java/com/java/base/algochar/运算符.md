
#### 比较 >> 和 >>> (无<<<)
```markdown
$ 正数而言，>>和>>>没区别

$ >> 表示右移, 为正时 高位补0, 为负时 高位补1
 20 >> 2 (0001 0100) => (0000 0101)=5        
-20 >> 2 (1110 1011) => (1110 1100)-右移2位-> (1111 1011)
          0111 1100
负数: 以补码的形式存储在内存中的
a =      111100
a =      001101
--------------------------------
(a | b)  111101 有一个是1就是1    全集
(a ^ b)  110001 都一样才为0       交集外  异或
(a & b)  001100 都为1才为1       交集
```
```markdown
＆	都是1，为1，否则为0	（A＆B），得到12，即0000 1100
|	都是0，为0，否则为1	（A | B）得到61，即 0011 1101
^	值相同，为0，否则为1	（A ^ B）得到49，即 0011 0001
〜	即0变成1，1变成0。	（〜A）得到-61，即1100 0011
<< 	按位左移运算符。左操作数按位左移右操作数指定的位数。	A << 2得到240，即 1111 0000
>> 	按位右移运算符。左操作数按位右移右操作数指定的位数。	A >> 2得到15即 1111
>>> 按位右移补零操作符。左操作数的值按右操作数指定的位数右移，移动得到的空位以零填充。
```

```markdown
最高位为1代表负数，最高位为0代表正数。
原码: 
反码: 原码按位取反 
补码: 反码+1
```
