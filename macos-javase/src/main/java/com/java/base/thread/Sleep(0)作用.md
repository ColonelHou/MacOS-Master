
> CPU竞争策略
> Unix是时间片算法，而Windows则属于抢占式的
```aidl
$ 十个人(10个进程)用刀叉(CPU)吃蛋糕(源源不断时间片), 

Unix是每人分配1分钟上来吃, 时间到了就换下一个

Window是每次计算优先级; 

sleep()函数干啥的?
一个人吃饱了, 给OS说在未来xx时间内我不吃了
(在未来的多少毫秒内我不参与CPU竞争)
Q1: sleep(1000)时间到了线程会不会被唤醒吗?
A1: 不一定, 只是告诉OS未来1秒内不参与竞争, 
    1秒后另一个线程正在使用cpu, 此时os不会重新分配 至到另一个线程over
Resume函数:用来唤醒挂起的线程


Q2: sleep(0)有意义吗？
A1: 有, 只是告诉未来0秒我不参与cpu竞争, 给其他线程一个机会


```
