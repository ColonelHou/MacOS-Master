0. 公平锁: 多个线程按照申请锁的顺序来获取锁
0. 非公平锁: 多个线程获取锁的顺序并不是按照申请锁的顺序
   非公平锁并高效率高, 因为公平锁上下文切换次数多
   
1. 悲观锁: 共享数据并发操作中, 一定会发生修改, 即使不修改也认为会修改; 适合写多场景
2. 乐观锁: 认为不加锁操作也应该没事的; eg:CAS
3. 独享锁: ReentrantLock就是独享, ReentrantReadWriteLock读共享,写独享; synchronized
4. 共享锁: 独享锁与共享锁也是通过AQS来实现的
5. 可重入锁(递归锁): 一定程序避免死锁

# 偏向锁、轻量级锁、重量级锁
> 这三种针对Synchronized
1. 偏向锁:同步代码一直被一个线程所访问，那么该线程会自动获取锁
2. 轻量级锁: 锁是偏向锁的时候，被另一个线程所访问，偏向锁就会升级为轻量级锁
            其他线程自旋形式尝试获取
3. 重量级锁: 锁为轻量级锁的时候，另一个线程虽然是自旋，但自旋不会一直持续下去，
            当自旋一定次数的时候，还没有获取到锁，就会进入阻塞，该锁膨胀为重量级锁
